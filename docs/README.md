# Documentação

Este diretório centraliza guias e referências da aplicação. Utilize os arquivos aqui para registrar decisões de produto, requisitos técnicos, fluxos de uso e materiais de apoio.

## Índice
- `project-overview.md`: resumo completo do contexto, objetivos e resultados obtidos até o momento.
- `architecture.md`: visão detalhada dos componentes do front-end, APIs e camada de dados.
- `mlops-llmops.md`: pipelines de dados, automações de treinamento, monitoramento e governança de prompts.
- `case-studies.md`: estudos de caso, datasets e métricas relacionados aos notebooks.

Adicione novos documentos conforme surgirem tutoriais, guias de API ou relatórios.
